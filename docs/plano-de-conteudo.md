# Plano de conteúdo

## Fase 1: Raças / Species

Objetivo:
Definir a base de trabalho para species jogáveis compatíveis com o fluxo do Foundry VTT e do sistema `dnd5e`.

Critérios de pronto:
- Convenções de nomenclatura revisadas.
- Tipo de item correto validado no sistema `dnd5e`.
- Processo seguro de criação e revisão no Foundry confirmado.

O que não fazer ainda:
- Não criar subclasses.
- Não criar conteúdo em massa.
- Não presumir campos nativos sem validação.

## Fase 2: Talentos raciais

Objetivo:
Organizar os talentos associados às species já validadas na fase anterior.

Critérios de pronto:
- Relação entre species e talentos raciais documentada.
- Fluxo de criação no Foundry repetível e revisado.
- Estrutura de nomes e labels mantida com consistência.

O que não fazer ainda:
- Não avançar para classes sem validar os talentos raciais.
- Não transformar material autoral em schema nativo sem confirmação.
- Não criar subclasses.

## Fase 3: Talentos / Features

Objetivo:
Preparar a camada de features gerais do módulo com organização compatível com `dnd5e`.

Critérios de pronto:
- Critérios de classificação de features definidos.
- Processo de revisão funcional registrado.
- Nomenclatura interna consistente com as fases anteriores.

O que não fazer ainda:
- Não criar classes antes da validação desta fase.
- Não editar compendiums manualmente fora do Foundry.
- Não criar subclasses.

## Fase 4: Backgrounds

Objetivo:
Planejar e validar a estrutura de backgrounds dentro da ordem do projeto.

Critérios de pronto:
- Decisão tomada sobre pack próprio de backgrounds antes de qualquer alteração no manifesto.
- Tipo de item e comportamento esperado confirmados no `dnd5e`.
- Regras de consistência com species, talentos raciais e features documentadas.

O que não fazer ainda:
- Não alterar `module.json` sem autorização.
- Não criar banco de compendium manualmente.
- Não criar subclasses.

## Fase 5: Classes

Objetivo:
Estruturar classes somente após a base anterior estar estável e validada.

Critérios de pronto:
- Fluxo de advancements e organização de classe compreendidos no sistema.
- Processo de revisão testado com escopo pequeno.
- Dependências com fases anteriores registradas.

O que não fazer ainda:
- Não criar subclasses antes da validação completa de classes.
- Não criar conteúdo em massa.
- Não inventar caminhos de dados do Foundry.

## Fase 6: Subclasses

Objetivo:
Trabalhar subclasses de forma controlada, uma por vez, sobre classes já validadas.

Critérios de pronto:
- Classes base previamente validadas.
- Escopo de cada subclasse definido individualmente.
- Processo de revisão funcional registrado para cada nova entrada.

O que não fazer ainda:
- Não criar várias subclasses em paralelo.
- Não iniciar subclasses sem base anterior consolidada.
- Não assumir automações complexas sem validação no Foundry.
